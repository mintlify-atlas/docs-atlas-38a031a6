This guide covers detailed production environment configuration for NutriFit. Follow these instructions to ensure optimal performance, security, and reliability.

## Environment File Configuration

### Creating Production .env

Start with the example environment file:

```bash
cp .env.example .env
```

Then customize for production:

<CodeGroup>
```bash Complete Production .env
# Application
APP_NAME="NutriFit"
APP_ENV=production
APP_KEY=base64:YOUR_GENERATED_KEY_HERE
APP_DEBUG=false
APP_URL=https://yourdomain.com
APP_LOCALE=es
APP_FALLBACK_LOCALE=es

# Security
BCRYPT_ROUNDS=12

# Logging
LOG_CHANNEL=stack
LOG_STACK=daily
LOG_LEVEL=error
LOG_DEPRECATIONS_CHANNEL=null

# Database (MySQL)
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=nutrifit
DB_USERNAME=nutrifit_user
DB_PASSWORD=your_secure_database_password

# Session
SESSION_DRIVER=database
SESSION_LIFETIME=120
SESSION_ENCRYPT=false
SESSION_PATH=/
SESSION_DOMAIN=.yourdomain.com

# Cache
CACHE_STORE=database
# Or use Redis for better performance:
# CACHE_STORE=redis

# Queue
QUEUE_CONNECTION=database
# Or use Redis for better performance:
# QUEUE_CONNECTION=redis

# Mail (Production SMTP)
MAIL_MAILER=smtp
MAIL_HOST=smtp.yourdomain.com
MAIL_PORT=587
MAIL_USERNAME=noreply@yourdomain.com
MAIL_PASSWORD=your_mail_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS="noreply@yourdomain.com"
MAIL_FROM_NAME="${APP_NAME}"

# Redis (Optional but recommended)
# REDIS_CLIENT=phpredis
# REDIS_HOST=127.0.0.1
# REDIS_PASSWORD=null
# REDIS_PORT=6379

# Admin credentials
ADMIN_PASSWORD=your_secure_admin_password
```

```bash Minimal Production .env
# Minimal configuration for quick setup
APP_NAME="NutriFit"
APP_ENV=production
APP_KEY=base64:GENERATED_KEY
APP_DEBUG=false
APP_URL=https://yourdomain.com

DB_CONNECTION=mysql
DB_DATABASE=nutrifit
DB_USERNAME=nutrifit_user
DB_PASSWORD=secure_password

MAIL_MAILER=smtp

MAIL_PORT=587
MAIL_USERNAME=noreply@yourdomain.com
MAIL_PASSWORD=mail_password

QUEUE_CONNECTION=database
```
</CodeGroup>

<Warning>
  Always use strong, unique passwords for database and admin accounts. Never reuse development passwords in production.
</Warning>

## Database Setup

### MySQL Configuration

<Steps>
  <Step title="Create Database">
    Create a dedicated database for NutriFit:
    
    ```sql
    CREATE DATABASE nutrifit CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```
  </Step>

  <Step title="Create Database User">
    Create a dedicated user with appropriate permissions:
    
    ```sql
    CREATE USER 'nutrifit_user'@'localhost' IDENTIFIED BY 'secure_password';
    GRANT ALL PRIVILEGES ON nutrifit.* TO 'nutrifit_user'@'localhost';
    FLUSH PRIVILEGES;
    ```
    
    <Note>
      Replace `secure_password` with a strong password generated by a password manager.
    </Note>
  </Step>

  <Step title="Run Migrations">
    Execute database migrations:
    
    ```bash
    php artisan migrate --force
    ```
    
    The `--force` flag is required for production environments.
  </Step>

  <Step title="Seed Initial Data">
    Create admin user and initial data:
    
    ```bash
    php artisan db:seed
    ```
    
    Default admin credentials:
    - Email: `nutrifit2026@gmail.com`
    - Password: Set via `ADMIN_PASSWORD` in `.env`
  </Step>
</Steps>

### Database Optimization

<Tabs>
  <Tab title="MySQL Settings">
    Optimize MySQL for NutriFit workload:
    
    ```ini
    # /etc/mysql/my.cnf or /etc/my.cnf
    [mysqld]
    max_connections = 150
    innodb_buffer_pool_size = 1G
    innodb_log_file_size = 256M
    innodb_flush_log_at_trx_commit = 2
    query_cache_size = 0
    query_cache_type = 0
    ```
    
    Restart MySQL after changes:
    ```bash
    sudo systemctl restart mysql
    ```
  </Tab>

  <Tab title="Connection Pooling">
    Laravel uses persistent connections by default for MySQL. No additional configuration needed.
    
    For high-traffic scenarios, consider using ProxySQL or PgBouncer (for PostgreSQL).
  </Tab>

  <Tab title="Indexes">
    NutriFit migrations include optimized indexes. Verify they exist:
    
    ```sql
    SHOW INDEX FROM appointments;
    SHOW INDEX FROM attentions;
    SHOW INDEX FROM users;
    ```
    
    Key indexes:
    - `appointments.nutritionist_id`
    - `appointments.patient_id`
    - `appointments.status`
    - `appointments.scheduled_date`
  </Tab>
</Tabs>

## Mail Server Configuration

### SMTP Providers

<AccordionGroup>
  <Accordion title="Postmark" icon="envelope" defaultOpen>
    Professional transactional email service:
    
    ```bash
    MAIL_MAILER=smtp
    MAIL_HOST=smtp.postmarkapp.com
    MAIL_PORT=587
    MAIL_USERNAME=your_postmark_token
    MAIL_PASSWORD=your_postmark_token
    MAIL_ENCRYPTION=tls
    MAIL_FROM_ADDRESS="noreply@yourdomain.com"
    ```
    
    **Benefits:**
    - High deliverability rates
    - Detailed analytics
    - Bounce/spam handling
    - Free tier available
  </Accordion>

  <Accordion title="Amazon SES" icon="aws">
    Cost-effective for high volume:
    
    ```bash
    MAIL_MAILER=ses
    AWS_ACCESS_KEY_ID=your_key_id
    AWS_SECRET_ACCESS_KEY=your_secret_key
    AWS_DEFAULT_REGION=us-east-1
    MAIL_FROM_ADDRESS="noreply@yourdomain.com"
    ```
    
    **Benefits:**
    - Very low cost ($0.10 per 1,000 emails)
    - Scalable
    - Integrated with AWS ecosystem
  </Accordion>

  <Accordion title="Mailgun" icon="envelope-open">
    Developer-friendly email service:
    
    ```bash
    MAIL_MAILER=mailgun
    MAILGUN_DOMAIN=yourdomain.com
    MAILGUN_SECRET=your_api_key
    MAIL_FROM_ADDRESS="noreply@yourdomain.com"
    ```
    
    Requires package:
    ```bash
    composer require symfony/mailgun-mailer
    ```
  </Accordion>

  <Accordion title="Self-Hosted SMTP" icon="server">
    Use your own mail server:
    
    ```bash
    MAIL_MAILER=smtp
    MAIL_HOST=mail.yourdomain.com
    MAIL_PORT=587
    MAIL_USERNAME=noreply@yourdomain.com
    MAIL_PASSWORD=your_password
    MAIL_ENCRYPTION=tls
    ```
    
    <Warning>
      Ensure proper SPF, DKIM, and DMARC records are configured to prevent emails from being marked as spam.
    </Warning>
  </Accordion>
</AccordionGroup>

## Cache Configuration

### Database Cache (Default)

NutriFit uses database caching by default:

```bash
CACHE_STORE=database
```

Ensure cache table exists:

```bash
php artisan cache:table
php artisan migrate
```

### Redis Cache (Recommended)

For better performance, use Redis:

<Steps>
  <Step title="Install Redis">
    ```bash
    # Ubuntu/Debian
    sudo apt install redis-server
    
    # Start Redis
    sudo systemctl start redis
    sudo systemctl enable redis
    ```
  </Step>

  <Step title="Install PHP Redis Extension">
    ```bash
    sudo apt install php8.2-redis
    sudo systemctl restart php8.2-fpm
    ```
  </Step>

  <Step title="Configure Laravel">
    Update `.env`:
    ```bash
    CACHE_STORE=redis
    SESSION_DRIVER=redis
    QUEUE_CONNECTION=redis
    
    REDIS_CLIENT=phpredis
    REDIS_HOST=127.0.0.1
    REDIS_PASSWORD=null
    REDIS_PORT=6379
    ```
  </Step>

  <Step title="Test Redis Connection">
    ```bash
    php artisan tinker
    ```
    
    ```php
    Cache::put('test', 'value', 60);
    Cache::get('test'); // Should return 'value'
    ```
  </Step>
</Steps>

### Cache Optimization

Clear and recache for optimal performance:

```bash
# Clear all caches
php artisan optimize:clear

# Rebuild caches
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

<Note>
  Run these commands after every deployment to ensure caches are up to date.
</Note>

## Security Configuration

### Application Security

<AccordionGroup>
  <Accordion title="Environment Security" icon="lock">
    Protect sensitive configuration:
    
    ```bash
    # Set restrictive permissions
    chmod 600 .env
    
    # Ensure proper ownership
    chown www-data:www-data .env
    ```
    
    Never commit `.env` to version control:
    ```bash
    # .gitignore already includes
    .env
    .env.backup
    .env.production
    ```
  </Accordion>

  <Accordion title="Directory Permissions" icon="folder-lock">
    Set correct filesystem permissions:
    
    ```bash
    # Storage and cache directories
    chmod -R 755 storage bootstrap/cache
    chown -R www-data:www-data storage bootstrap/cache
    
    # Application files (read-only)
    chmod -R 644 app/ config/ routes/
    chmod -R 755 artisan
    ```
  </Accordion>

  <Accordion title="SSL/TLS Configuration" icon="shield-check">
    Force HTTPS for all requests. Add to `.env`:
    
    ```bash
    APP_URL=https://yourdomain.com
    ```
    
    Configure in `app/Http/Middleware/TrustProxies.php` if behind proxy:
    ```php
    protected $proxies = '*';
    protected $headers = Request::HEADER_X_FORWARDED_ALL;
    ```
  </Accordion>

  <Accordion title="Rate Limiting" icon="gauge-high">
    NutriFit includes rate limiting. Configure in `config/fortify.php`:
    
    ```php
    'limiters' => [
        'login' => 'login',
        'two-factor' => 'two-factor',
    ],
    ```
    
    Default: 5 login attempts per minute per IP.
  </Accordion>
</AccordionGroup>

## Logging Configuration

### Production Logging

Configure appropriate log level:

```bash
LOG_CHANNEL=stack
LOG_STACK=daily
LOG_LEVEL=error
```

This creates daily log files in `storage/logs/`.

### Log Rotation

Configure automatic log rotation:

```php
// config/logging.php
'daily' => [
    'driver' => 'daily',
    'path' => storage_path('logs/laravel.log'),
    'level' => env('LOG_LEVEL', 'error'),
    'days' => 14, // Keep logs for 14 days
],
```

## Performance Tuning

### OPcache Configuration

Enable OPcache for PHP (should be enabled by default):

```ini
; /etc/php/8.2/fpm/php.ini
opcache.enable=1
opcache.memory_consumption=256
opcache.interned_strings_buffer=16
opcache.max_accelerated_files=10000
opcache.revalidate_freq=2
```

Restart PHP-FPM:

```bash
sudo systemctl restart php8.2-fpm
```

### PHP-FPM Tuning

Optimize PHP-FPM pool:

```ini
; /etc/php/8.2/fpm/pool.d/www.conf
pm = dynamic
pm.max_children = 50
pm.start_servers = 5
pm.min_spare_servers = 5
pm.max_spare_servers = 35
pm.max_requests = 500
```

### Laravel Optimizations

Run all optimization commands:

```bash
# Install production dependencies only
composer install --optimize-autoloader --no-dev

# Cache everything
php artisan config:cache
php artisan route:cache
php artisan view:cache

# Build optimized assets
npm run build
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="500 Internal Server Error">
    1. Check Laravel logs: `storage/logs/laravel.log`
    2. Verify file permissions
    3. Clear all caches: `php artisan optimize:clear`
    4. Check web server error logs
  </Accordion>

  <Accordion title="Emails Not Sending">
    1. Verify SMTP credentials
    2. Check queue worker is running
    3. Review failed jobs: `php artisan queue:failed`
    4. Test mail configuration with tinker
  </Accordion>

  <Accordion title="Database Connection Failed">
    1. Verify MySQL is running: `sudo systemctl status mysql`
    2. Check credentials in `.env`
    3. Test connection: `mysql -u nutrifit_user -p nutrifit`
    4. Verify MySQL grants are correct
  </Accordion>
</AccordionGroup>
